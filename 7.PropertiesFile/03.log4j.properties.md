# What is log4j.properties?

log4j.properties is a configuration file for Log4j (logging framework).

Logging means:

Recording what your application is doing (info, warnings, errors, debug messages).

Log4j helps you:

Print logs to the console

Save logs to files

Control log level (INFO, DEBUG, ERROR, etc.)

Decide log format

** Log4j removes the need to use System.out.println().


**Logger** : Decides whether a log should be printed

**Appender**: Decides where logs go

**Layout**: Decides how logs look

## 1️⃣ Root Logger
```
log4j.rootLogger = INFO, stdout, file
```
The root logger is the main logger for your entire application.

Every log message eventually goes through the root logger unless overridden.

Defines the global logging behavior

Applies to all classes by default

INFO → Minimum log level

stdout → Send logs to console

file → Send logs to a file

| Level | Description             |
| ----- | ----------------------- |
| DEBUG | Detailed debugging      |
| INFO  | Normal application flow |
| WARN  | Something unusual       |
| ERROR | Something failed        |
| FATAL | Application crash       |

## 2️⃣ File Appender (Log to File)

An appender decides where logs go.

```
log4j.appender.file=org.apache.log4j.FileAppender
```

Creates an appender that writes logs to a file.

**Log File Location**
```
log4j.appender.file.File=target/logs/Default.log
```

Path of the log file.

Keeps logs for later review

Useful for debugging failed tests

**Immediate Flush**

log4j.appender.file.ImmediateFlush=true

Writes logs to file immediately.

Ensures logs are not lost if tests crash

⚠️ Slightly slower, but safer

## Append Mode
```
log4j.appender.file.Append=true
```
Controls whether logs:

Append to existing file

Or overwrite it

| Value   | Behavior                   |
| ------- | -------------------------- |
| `true`  | Add logs to existing file  |
| `false` | Overwrite file on each run |


### Threshold (Commented)

#log4j.appender.file.Threshold=debug

Sets log level only for this appender.

If enabled:

File would include DEBUG logs

Console would still show INFO+

## 3️⃣ Log Format (Layout)

```
log4j.appender.file.layout=org.apache.log4j.PatternLayout
log4j.appender.file.layout.ConversionPattern=%d{yyyy-MM-dd HH:mm:ss.SSS} %-5p %c{1} - %m%n

```

Defines how each log line looks.

| Part      | Meaning                 |
| --------- | ----------------------- |
| `%d{...}` | Timestamp               |
| `%-5p`    | Log level (INFO, ERROR) |
| `%c{1}`   | Class name              |
| `%m`      | Log message             |
| `%n`      | New line                |

**sample output:** 2026-01-20 10:15:32.456 INFO  LoginTest - User logged in successfully

##  Console Appender (stdout)

This section prints logs to terminal / IDE console.

```
log4j.appender.stdout=org.apache.log4j.ConsoleAppender
```

Real-time visibility during test execution

Easy debugging

**console target**

```
log4j.appender.stdout.Target=System.out
```

Sends logs to standard output (System.out).

**consloe layout**

```
log4j.appender.stdout.layout=org.apache.log4j.PatternLayout
log4j.appender.stdout.layout.ConversionPattern=...
```
Same format as file

Makes debugging easier (consistent logs)


| Section          | Purpose               |
| ---------------- | --------------------- |
| Root Logger      | Sets global log level |
| File Appender    | Writes logs to file   |
| Console Appender | Shows logs in console |
| Layout           | Defines log format    |
| Append           | Keep old logs         |
| Immediate Flush  | Write logs instantly  |
