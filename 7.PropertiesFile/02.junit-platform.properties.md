# What is junit-platform.properties?

junit-platform.properties is a configuration file used by JUnit 5 (JUnit Platform).

When you run tests, JUnit automatically:

Reads this file

Applies the settings to all test executions

üëâ In a Cucumber project, this file controls:

How Cucumber tests run

How reports are generated

Whether tests run in parallel

Since Cucumber runs on top of JUnit Platform, this file controls how Cucumber executes tests, even though the file is not named cucumber.properties.

```
Your Test Code (Cucumber)
        ‚Üì
JUnit Platform (Runner Engine)
        ‚Üì
Threading, Parallelism, Reports

```

# Why does Cucumber use this file?

In JUnit 5, execution settings (parallelism, reporting, etc.) are not handled in runner classes, but via configuration files.

‚úÖ Cucumber integrates into JUnit Platform

‚úÖ JUnit Platform reads junit-platform.properties automatically

‚úÖ Cucumber picks up those values

### 1Ô∏è‚É£ Cucumber Execution Mode
```

```

Controls whether Cucumber:

Only checks step definitions, OR

Actually executes test steps

| Value   | Meaning                          |
| ------- | -------------------------------- |
| `true`  | Only checks steps (no execution) |
| `false` | Executes tests normally          |

Plugins tell Cucumber how to show and save results.

### Plugin Breakdown

üîπ pretty

üîπ summary

üîπ timeline:target/cucumber-results/main/cucumber-timeline

üîπ html:target/cucumber-results/main/report.html

üîπ rerun:target/cucumber-results/main/rerun.txt

Lets you re-run failures quickly

### 6Ô∏è‚É£ Parallel Execution Strategy


Defines how thread count is decided.

| Strategy  | Description             |
| --------- | ----------------------- |

| `fixed`   | Fixed number of threads |
| `custom`  | User-defined logic      |

# Rerunning only failed scenarios

1. first run

```
cucumber.plugin=pretty, summary, rerun:target/cucumber-results/main/rerun.txt
```

2.Create a second runner class to rerun:

```

@CucumberOptions(
  features = "@target/cucumber-results/main/rerun.txt",
  plugin = {
    "pretty",
    "html:target/cucumber-results/rerun/report.html"
  }
)
public class RerunFailedTests { }
```

3.Execute RerunFailedTests to just run the failures.
